# Cloud Resume Challenge — Ali Ahmed

A cloud-native CV built on AWS, following the [Cloud Resume Challenge](https://cloudresumechallenge.dev/). Live at **[ali-ahmed.me](https://ali-ahmed.me)**.

---

## Architecture

```
                        ┌─────────────┐
                        │  Route 53   │
                        │  (DNS)      │
                        └──────┬──────┘
                               │
                        ┌──────▼──────┐
                        │ CloudFront  │
                        │ (HTTPS/CDN) │
                        └──────┬──────┘
                               │
                    ┌──────────▼──────────┐
                    │       S3 Bucket     │
                    │     (index.html)    │
                    └─────────────────────┘

         JavaScript calls API Gateway for:

    ┌─────────────────┐      ┌─────────────────┐
    │  Visitor Counter│      │  Contact Form   │
    │  GET /count     │      │  POST /contact  │
    └────────┬────────┘      └────────┬────────┘
             │                        │
    ┌────────▼────────┐      ┌────────▼────────┐
    │     Lambda      │      │     Lambda      │
    │ visitor_counter │      │  contact_form   │
    └────────┬────────┘      └────────┬────────┘
             │                        │
    ┌────────▼────────┐      ┌────────▼────────┐
    │    DynamoDB     │      │      SES        │
    │ (visitor count) │      │ (email delivery)│
    └─────────────────┘      └─────────────────┘
```

## AWS Services Used

| Service | Purpose |
|---|---|
| S3 | Static website hosting |
| CloudFront | HTTPS, CDN, caching |
| Route 53 | Custom domain DNS |
| ACM | SSL/TLS certificate |
| Lambda | Serverless functions (Python 3.12) |
| DynamoDB | Visitor counter storage |
| API Gateway | Public HTTP endpoints |
| SES | Contact form email delivery |
| SAM | Infrastructure as Code |
| CloudFormation | Stack deployment |
| IAM | Permissions and roles |

---

## Features

- **Live CV** at a custom domain with HTTPS
- **Visitor counter** — serverless, increments on every visit and displays count
- **Contact form** — sends email directly to inbox via AWS SES
- **Infrastructure as Code** — all backend resources defined in `template.yaml`
- **CI/CD pipeline** — GitHub Actions automatically deploys on every push to `main`

## Infrastructure as Code

All backend infrastructure is defined in `template.yaml` using AWS SAM:

- **VisitorCountFunction** — Lambda + API Gateway GET /count
- **VisitorCountTable** — DynamoDB table
- **ContactFormFunction** — Lambda + API Gateway POST /contact
- IAM roles and permissions auto-generated by SAM

Deploy with:

```bash
sam build
sam deploy
```

---

## CI/CD Pipeline

GitHub Actions workflow (`.github/workflows/deploy.yml`) runs on every push to `main`:

1. Checkout code
2. Configure AWS credentials
3. Build SAM application
4. Deploy SAM stack to AWS
5. Upload `index.html` to S3
6. Invalidate CloudFront cache
